# 项目简介

基于 HarmonyOS ArkTS 的校园食堂提前点餐应用。9 个档口浏览点餐，用户注册/登录（密码 SHA-256 哈希存储），购物车弹窗实时增减，跨店自动拆分订单，下单确认弹窗，取餐码管理。

兼容 DevEco Studio 5.1.1 ~ 6.1.1。

---

## 功能详情

### 用户系统
- **注册**：手机号 + 密码，手机号校验格式（1[3-9]xxxxxxxxx），密码强度 ≥6 位
- **登录**：手机号 + 密码，SHA-256 哈希比对
- **密码修改**：原密码哈希验证，新密码哈希存储
- 退出登录、个人资料编辑

### 点餐流程
- 首页 9 个档口（土豆片夹馍、扬州干拌面、山东杂粮煎饼等）
- 分类筛选（面食/米饭套餐/小吃加料/特色推荐）
- 关键词搜索
- 商品页 `+/-` 增减数量，**点数字弹出输入框**可自由输入（上限 99）
- 底部橙色购物车栏实时显示总数与总价（含 ¥1 服务费）

### 购物车弹窗
- 点底部橙色栏弹出半屏购物车
- 商品清单可 `+/-` 调数量、✕ 删除、点数字编辑
- 「清空」一键清空购物篮
- 「直接下单」按钮 → 确认弹窗 → 提交

### 订单系统
- 提交时**按档口自动拆分**：不同店的商品各自生成独立订单
- 每笔订单生成随机取餐码 + 防冲突 ID
- 订单筛选（全部/待取餐/已完成/已取消）
- 确认取餐 / 取消订单
- 「再来一单」先清空购物车再填入

### 安全机制
- 密码使用 **cryptoFramework SHA-256** 哈希存储，不可逆
- 软实现 SHA-256 回退（cryptoFramework 不可用时）
- 无明文密码存储或比较

## 技术架构

| 层 | 说明 |
|-----|------|
| UI | ArkUI 声明式，`@Component` + `@Builder` |
| 状态 | `@State` 管理视图切换和数据流 |
| 存储 | `preferences` 键值对本地持久化 |
| 密码 | `cryptoFramework.createMd('SHA256')` |
| 构建 | Hvigor |

## 快速开始

1. DevEco Studio 打开项目
2. 修改 `local.properties` 的 SDK 路径
3. `Build → Build Hap(s)`
4. 模拟器或真机 `Run`

## 项目结构

```
entry/src/main/ets/pages/Index.ets   ← 主文件（全部 UI + 业务逻辑）
entry/src/main/ets/entryability/     ← 应用入口
entry/src/main/ets/data/             ← 种子数据
entry/src/main/ets/utils/            ← 密码哈希、校验
entry/src/main/ets/models/           ← 类型定义
```
